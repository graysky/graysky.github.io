Bits Are Tricky

Challenges

  • Software engineering is different
    • Difficult to plan
    • Changes can ripple
    • Quality is hard
    • Scaling is hard
    • Change Becomes Difficult
  • Burden of code, assumptions, data, customers, etc
  • Reading & maintaining code is harder than writing code
  • Example: BlackBerry, Healthcare.gov, ClearCase
  • Reference: "How BlackBerry Blew It" about the downfall of Blackberry
  • Reference: "Dreaming in Code" about the Chandler project

Countermeasures

  • Must keep velocity up
  • Invest in automation
  • Use simple, 3rd party components when possible

Requirements

Challenges

  • Difficult to specify requirements
    • Example: Your Projects ;-)
    • Know the least at the beginning of a project
    • Domain expert != software expert
  • Unknown requirements
    • On new project no one knows
    • Example: oneforty
  • Second System
    • New system has to have all features of previous system plus more

Countermeasures

  • Agile Methods (when problem space is known)
  • Lean Startup (when problem space is unknown)

People & Organizations

Challenges

  • Culture Matters
    • People in groups behave differently
    • How are people rewarded/punished?
    • How are decisions made?
    • Example: Microsoft
  • Problems Occur at Boundaries
    • Boundaries between groups, departments, companies
    • Communication/coordination overhead grows O(2n)
    • Example: ContentStudio

Countermeasures

  • Invest in people
  • Smaller teams, smaller projects
  • Clear decision owner (Directly Responsible Individual)

Software Business

Challenges

  • Typical business challenges to start & grow
  • Competing on global scale
  • Incubents frequently disrupted
  • Example: SnapMyLife

Countermeasures

  • Lean Startup / Build a MVP ("Minimally Viable Product")
  • Test assumptions quickly
  • Disrupt Yourself
